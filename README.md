# 📊 排序演算法視覺化大廳與效能分析報告

這是一個用於展示及學習經典「排序演算法」的互動式網頁專案，同時包含理論複雜度分析與實際程式碼效能的實驗報告。

## 一、 前言
本報告旨在探討與比較五種經典排序演算法的運作原理與效能差異。透過互動式網頁展示排序過程，並結合理論的時間複雜度分析，以及實際程式碼的數據模擬，觀察在不同資料規模（n）下各演算法的執行時間變化，藉此驗證理論並總結各排序法的適用情境。

---

## 二、 五個排序法介紹 (基本原理與操作)

本專案利用 JavaScript 實作了以下五種經典排序演算法：

1. **氣泡排序法 (Bubble Sort)**
   * **原理**：相鄰元素兩兩比較，若順序錯誤則交換，每回合將最大值像泡泡一樣推向未排序區間的最右側。
2. **選擇排序法 (Selection Sort)**
   * **原理**：將陣列分為已排序與未排序區間，每次從未排序區間中找出最小值，與該區間的第一個元素交換。
3. **插入排序法 (Insertion Sort)**
   * **原理**：如同整理撲克牌，邊從未排序區間取出元素，邊由右至左與已排序區間比較，並插入到正確的位置。
4. **合併排序法 (Merge Sort)**
   * **原理**：採用「分而治之 (Divide and Conquer)」策略，將陣列不斷對半分割至單一元素，再依序將兩個子陣列合併。
5. **快速排序法 (Quick Sort)**
   * **原理**：挑選基準點 (Pivot)，將小於基準點的元素移至左側，大於的移至右側，並對左右兩側遞迴執行。

## 三、 複雜度分析

以下為五種排序演算法在時間與空間複雜度上的理論分析：

| 排序演算法 | 最佳時間複雜度 | 平均時間複雜度 | 最差時間複雜度 | 空間複雜度 | 排序穩定性 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **氣泡排序** | O(n) | O(n²) | O(n²) | O(1) | 穩定 |
| **選擇排序** | O(n²) | O(n²) | O(n²) | O(1) | 不穩定 |
| **插入排序** | O(n) | O(n²) | O(n²) | O(1) | 穩定 |
| **合併排序** | O(n log n) | O(n log n) | O(n log n) | O(n) | 穩定 |
| **快速排序** | O(n log n) | O(n log n) | O(n²) | O(log n) | 不穩定 |

## 五、 結果呈現與效能比較

從實驗數據中可觀察到明顯的效能分水嶺：

1. **O(n²) 群組：** 當資料量 n 增加 10 倍時，氣泡、選擇、插入排序的耗時增加約 100 倍（符合平方增長）。其中氣泡排序因頻繁交換而最慢。
2. **O(n log n) 群組：** 合併與快速排序在處理 10 萬筆資料時仍保持在 30 毫秒以內，展現了高階演算法處理海量數據的絕對優勢。

## 六、 心得與結論

透過本次實作互動式網頁與效能測速，我對排序演算法有以下深刻體會：
1. **理論與實務的結合：** 實驗數據完美印證了 Big O 時間複雜度的理論。在面對龐大數據時，選擇正確的演算法是程式效能的關鍵。
2. **沒有絕對完美的排序法：** 快速排序在隨機資料下最快；合併排序雖然穩定但需要額外空間；插入排序則在資料接近排好時效率最高。
3. **結論：** 實際開發應根據資料規模與硬體限制，靈活選擇最適合的排序工具。
